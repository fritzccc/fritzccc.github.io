[搬运&更新]30天精通 RxJS (02)-Functional Programming 基本观念

发表于 2018-11-08 | 分类于 30天精通Rx ， Rx

Functional Programming 是 Rx 最重要的观念之一，基本上只要学会 FP 要上手 Rx 就不难了！Functional Programming 可以说是近年来的显学，各种新的函式编程语言推出之外，其他旧有的语言也都在新版中加强对 FP 的支援！

什么是 Functional Programming？

Functional Programming 是一种编程范式 (Programming Paradigm)，就像Object-oriented Programming (OOP) 一样，就是一种写程序的方法论，这些方法论告诉我们如何思考以及解决问题。

简单说 Functional Programming 的核心思想就是做运算处理，并用 function 来思考问题

例如以下的算术运算式

( 5 + 6 ) - 1 * 3

我们把每个运算区间包裹成一个个不同的 function ，并用这些 function 组合出我们想要的结果，这就是嘴嘴基本的 Functional Programming。

Functional Programming 基本要件

与OOP一样，不是所有的语言都支持FP，要能够支持FP的语言必须要符合【Function一等公民】的特性

例如JAVA里，只有名词是一等公民，而Method（Function）是 “二等公民”

Function 能够被赋值给变量

Function 能够被当做参数传入

Function 能够被当做返回值

Functional Programming 重要特性

Expression,no Statement

Functional Programming 都是表达式 (Expression) 而不是陈述式 (Statement)

表达式 是一个运算过程，一定会有返回值，例如执行一个 function

陈述式 则是表现某个行为，例如一个 赋值给一个变数

由于 Functional Programming 最早就是为了做运算处理不管 I/O，而 Statement 通常都属于对系统 I/O 的操作，所以 FP 很自然的不会是 Statement。

当然在实际操作中不可能完全没有 I/O 的操作，Functional Programming 只要求对 I/O 操作限制到最小，不要有不必要的 I/O 行为，尽量保持运算过程的单纯。

Pure Function

也就是纯函数，是指一个 function 只要被基于相同的参数，就永远会返回相同的结果，并且没有任何显著的副作用 (Side Effect)

可见 slice 不管执行几次，返回值都是相同的，并且除了返回一个值 (value) 之外并没有做任何事，所以 slice 就是一个 pure function。而 :

换作 splice，由于 splice 每执行一次就会影响 arr 的值，导致每次結果都不同，这就很明显不是一个 pure function。

Side Effect

Side Effect 是指一个 function 做了跟本身运算返回值无关的事，比如说修改某个全局变量，或是修改传入参数的值，甚至是执行 console.log 都算是 Side Effect。

Functional Programming 强调没有 Side Effect，也就是 function 要保持纯粹，只做运算并返回一个值，没有其他额外的行为。

这里列举几个前端常见的 Side Effect，但不是全部

  • 发送 http request

  • 在画面打印出值或是 log

  • 获取用户的 input

  • Query DOM 对象

Referential transparency

前面提到的 pure function 不管外部环境如何，只要参数相同，函式执行的返回结果必定相同。这种不依赖任何外部状态，只依赖于传入的参数的特性也称为 引用透明(Referential transparency)

Functional Programming 的优势

可读性高

易于维护

因为 Pure function 等特性，执行结果不依赖外部状态，且不会对外部环境有任何操作，使 Functional Programming 能更好的Debug及编写单元测试。

易于 併行/平行 处理

Functional Programming 易于做併行/平行(Concurrency/Parallel)处理，因为我们基本上只做运算不碰 I/O，再加上没有 Side Effect 的特性，所以较不用担心 deadlock 等问题。
